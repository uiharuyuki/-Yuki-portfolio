# コーディング指針

このファイルは、このポートフォリオサイトを編集するときの作業ルールです。
ユーザーの指示によって新しいルールが追加されたと判断した場合は、このファイルに反映します。
追加する内容は、細かい実装値ではなく大まかな方針に留めます。

## 同期ルール

- `AGENTS.md` と `CLAUDE.md` は同じ内容にします。
- どちらか片方を修正した場合は、もう片方にも同じ修正を反映します。
- 現在の作業環境では、2つのファイルをハードリンクにして同期します。

## HTML

- 基本構造は `header`、`main`、`footer` を使います。
- `header` と `footer` の中に、原則として `section` は作りません。
- `section` は `main` の中で、`About`、`Works`、`Contact` など意味のある本文の区切りに使います。
- `main` 内の本文エリアは、必要に応じて `section > .container` の形にします。
- `container` は本文の横幅を制限するために使います。ヘッダーとフッターには原則として適用しません。

## CSS

- クラスは最小限にします。構造だけで安全に指定できる場所はタグや子孫セレクタを使い、役割が明確な要素だけクラスを付けます。
- 色は直接指定せず、`assets/css/base.css` の `:root` にCSS変数として定義します。
- 共通パーツのCSSは `assets/css/common.css` に置きます。
- ページ固有のCSSは `assets/css/pages/` 配下に置きます。
- 画面全体に関わる装飾は、スクロールの影響を受けない共通要素として扱います。
- 固定表示の装飾は、既存要素と干渉しない余白を確保します。
- ファーストビューの背景演出は、固定枠と独立して画面全体を使えるようにします。

## Assets

- 画像アセットは用途に応じて `assets/images` 配下で整理します。

## Header

- ヘッダーはファーストビュー画像を背面に見せるため、画面上部に固定して重ねます。
- ナビゲーションは中央配置にし、GitHubリンクは右側に配置します。
- 現在ページのナビリンクには、状態が分かるクラスとアクセシビリティ属性を付けます。
