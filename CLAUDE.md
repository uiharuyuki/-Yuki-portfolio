# コーディング指針

このファイルは、このポートフォリオサイトを編集するときの作業ルールです。
ユーザーの指示によって新しいルールが追加されたと判断した場合は、このファイルに反映します。
追加する内容は、細かい実装値ではなく大まかな方針に留めます。

## 同期ルール

- `AGENTS.md` と `CLAUDE.md` は同じ内容にします。
- どちらか片方を修正した場合は、もう片方にも同じ修正を反映します。
- 現在の作業環境では、2つのファイルをハードリンクにして同期します。

## README

- サイト構成、データ管理、ビルド手順に関わる変更をした場合は、`README.md` も同時に更新します。
- READMEには、編集時に迷いやすい元データの場所と生成手順を優先して記載します。

## HTML

- 基本構造は `header`、`main`、`footer` を使います。
- `header` と `footer` の中に、原則として `section` は作りません。
- `section` は `main` の中で、`About`、`Works`、`Contact` など意味のある本文の区切りに使います。
- `main` 内の本文エリアは、必要に応じて `section > .container` の形にします。
- `container` は本文の横幅を制限するために使います。ヘッダーとフッターには原則として適用しません。
- 共通ヘッダーと共通フッターは、複数ページで使う前提のリンク設計にします。
- Worksカードの表示内容は `data/works.json` を正とし、ビルド時に生成します。トップページは `featured` を付けた作品のみ、Works一覧ページは全作品を表示し、どちらも `category` ごとにグループ分けします（カテゴリが1種類のときはフラットな一覧）。
- WorksカードはトップページとWorks一覧ページで表示構造を揃え、ジャンル、作品タイトル、使用技術、説明文の順で情報を整理します。
- Worksカードの外部リンクは、Works詳細ページにも同じ導線として表示します。
- WorksカードはWeb以外の作品にも対応します。`images` の `mobile` を省くと1枚画像カードになり、メタ見出しは `metaLabel`（既定「使用技術」）で変更できます。
- Works詳細ページの本文は `content/works/` 配下のMarkdownを正とし、構造化された基本情報は `data/works.json` に置きます。
- Worksの一覧用要約と詳細本文は役割を分け、使用技術などの基本情報は二重管理しません。
- Aboutプロフィールの共通表示内容は `data/about.json` を正とし、トップページとAboutページはビルド時に同じデータから生成します。
- 生成対象のHTMLは直接編集せず、元データを修正して `npm run build` で反映します。

## CSS

- クラスは最小限にします。構造だけで安全に指定できる場所はタグや子孫セレクタを使い、役割が明確な要素だけクラスを付けます。
- 色は直接指定せず、`assets/css/base.css` の `:root` にCSS変数として定義します。
- 複数箇所で揃える主要な見出し（ページ／セクションタイトル）のフォントサイズは、`:root` のCSS変数で一元管理します。
- 共通パーツのCSSは `assets/css/common.css` に置きます。
- 複数ページで使うカードやボタンなどの部品は `assets/css/component.css` に置きます。
- ページ固有のCSSは `assets/css/pages/` 配下に置きます。
- 画面全体に関わる装飾は、スクロールの影響を受けない共通要素として扱います。
- 固定表示の装飾は、既存要素と干渉しない余白を確保します。
- ファーストビューの背景演出は、固定枠と独立して画面全体を使えるようにします。
- 画面幅に応じて可変する余白やサイズには `clamp()` を使います。基準となる画面幅は **480px（最小）〜 1080px（最大）** の範囲で設計します。

## Assets

- 画像アセットは用途に応じて `assets/images` 配下で整理します。
- OGPやSNS共有用などページ表示以外の共通画像は、用途別ディレクトリに分けて管理します。
- `assets/images/icons` では、favicon、Apple touch icon、PWA用アイコンを用途別に管理します。
- 画像は表示用途に合わせて解像度・形式・容量を調整し、使用しない元画像や旧画像は残しません。

## Header

- ヘッダーはファーストビュー画像を背面に見せるため、画面上部に固定して重ねます。
- ナビゲーションは中央配置にし、右端に `.header-end`（テーマトグル・リンク群・バーガーボタン）をまとめて配置します。
- 現在ページのナビリンクには、状態が分かるクラスとアクセシビリティ属性を付けます。

## ダークモード

- `[data-theme="dark"]` 属性を `<html>` に付与してテーマを切り替えます。
- OS設定をデフォルトとし、手動切り替えは `localStorage` に保存して次回以降も引き継ぎます。
- FOUC（テーマのちらつき）を防ぐため、各ページの `<head>` 先頭にインラインスクリプトを置きます。
- 色は `base.css` の `:root`（ライト）と `[data-theme="dark"]`（ダーク）にCSS変数で定義します。

## ローディング画面

- 初回訪問のみ表示し、`sessionStorage` で管理します（同一セッション内の2回目以降はスキップ）。
- 初回訪問時も保存テーマまたはOS設定に合わせてテーマを適用します。
- `assets/js/loading.js` がオーバーレイの表示制御を担います。
- ローディング用画像は `assets/images/loading/` 配下で管理します。

## カスタムカーソル

- デスクトップのみ（`(hover: hover) and (pointer: fine)` メディアクエリで判定）。
- `header.js` の `SiteHeader` コンポーネント内で生成・管理します。

## メール

- 公開ページのメールアドレスはHTMLに直書きせず、`[at]` 表記で記述し、`assets/js/email.js` で `@` に戻して表示・mailto・コピーに使います（スパムボット対策）。
